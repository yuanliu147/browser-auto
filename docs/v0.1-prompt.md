## 1. 项目背景与差异化定位

我要做一个开源的浏览器自动化工具 **browser-auto**。底层执行参照 Stagehand 的成熟方案(`act` / `extract` / `observe`),没必要重新发明。但项目的差异化目标在 **三件事**:

1. **分层记忆 (Memory Hierarchy)** —— 不是单一缓存,而是 L0~L3 四层记忆,粒度递减、复用递增、LLM 成本递减。
2. **架构缝隙里的扩展面** —— 后续 v0.5 / v1.0 要长出 Stagehand 长不出的中后台组件适配、可视化编排、动作策略、Daemon 长驻等能力,v0.1 必须把对应的 interface 焊好。
3. **本地优先 + 中文 LLM 一等公民** —— 默认零外部云依赖,提示词模板兼顾 OpenAI 和 DeepSeek。

**v0.1 不实现** 上述任何上层特性,但要把扩展口子全部焊进底层。下面会逐条点明哪些设计是"为后面留的口子"。

## 4. 分层记忆模型 (核心设计)

**这是 browser-auto 区别于 Stagehand 的主要技术特征。** 必须在 v0.1 落地。

### 4.1 四层模型

| 层                         | 记的是什么               | LLM 角色       | 适用                     | 命中开销            |
| -------------------------- | ------------------------ | -------------- | ------------------------ | ------------------- |
| **L3** ActionMemory(路径)  | 完整 ActionStep[]        | 不调           | act 操作                 | ~0(Playwright 重放) |
| **L2** LocatorMemory(节点) | 语义 → selector 候选链   | 仅校验 / 不调  | extract 字段、act 单点击 | 1 次 selector 试探  |
| **L1** RegionMemory(区域)  | 语义 → 子树入口 selector | 仅在子树内推理 | 复杂 extract、列表项     | LLM token 大幅压缩  |
| **L0** 无记忆              | —                        | 全页推理       | 首次 / 大改版            | 最贵                |

### 4.2 命中流程是降级链

```
extract({title})
  → L2 命中 ".product h2" → page.textContent() → 完事(0 LLM)
  → L2 失效 / 取不到值
  → L1 命中 region ".product" → LLM 在子树内重找 → 写回 L2
  → L1 失效
  → L0 全页 → 写回 L1+L2

act("提交订单")
  → L3 命中 plan → 重放(0 LLM)
  → L3 失效
  → L2 命中"提交按钮" selectors → 直接 click → 写回 L3
  → L0 全页规划
```

### 4.3 v0.1 实现范围

- ✅ **实现 L3 ActionMemory** —— act 路径缓存,最高优先级
- ✅ **实现 L2 LocatorMemory** —— extract 节点记忆,本质是"语义→selector"KV
- 🔌 **预留 L1 RegionMemory 接口,不实现** —— 接口摆着,落地放 v0.3
- 🔌 **L2 用于 act 也是 v0.5+** —— v0.1 act 只走 L3 / L0

### 4.4 接口与文件 schema

#### 4.4.1 统一 Memory 接口

```ts
interface Memory {
  // L3: act 整套路径
  recallPlan(instruction: string, fp: string): ActionStep[] | null;
  rememberPlan(instruction: string, fp: string, plan: ActionStep[]): void;
  invalidatePlan(instruction: string, fp: string): void;

  // L2: 单节点 selector
  recallLocator(semanticKey: string, fp: string): LocatorEntry | null;
  rememberLocator(semanticKey: string, fp: string, entry: LocatorEntry): void;
  invalidateLocator(semanticKey: string, fp: string): void;

  // L1: 占位接口,v0.1 抛 NotImplemented
  recallRegion(semanticKey: string, fp: string): RegionEntry | null;
  rememberRegion(semanticKey: string, fp: string, entry: RegionEntry): void;
}

type LocatorEntry = {
  selectors: string[]; // fallback 候选
  hint?: string; // 自然语言描述
  selectorHits: number[]; // 每个候选命中次数
  successCount: number;
  failCount: number;
};

type RegionEntry = {
  rootSelector: string;
  hint?: string;
};
```

```jsonc
{

    {
```

## 5. 核心 API 设计(锁定)

**双入口:Session 主入口,wrap 快捷方式。**

### 5.1 主入口:`createSession`

```ts
import { chromium } from "playwright";
import { createSession } from "browser-auto";

const session = await createSession({
  // 浏览器三种模式
  browser: { mode: "launch", engine: chromium, options: { headless: false } },
  // 或 { mode: 'attach', cdpEndpoint: 'http://localhost:9222' }
  // 或 { mode: 'persistent', userDataDir: './.profile' }

  llm: {
    provider: "openai", // 'openai' | 'anthropic' | 'deepseek'
    model: "gpt-4o-mini",
    apiKey: process.env.OPENAI_API_KEY,
  },

  memory: {
    enabled: true,
    path: "./.browser-auto/memory.json",
  },

  // v0.1 留接口,不实现
  policies: [], // BeforeActionInterceptor[]
  trace: undefined, // TraceSink
});

const page = await session.newPage();
await page.goto("https://example.com");
await page.act("点击右上角的登录按钮");
await page.act("在用户名输入框填入 ${username}", { username: "alice" });
const data = await page.extract({ schema: z.object({ title: z.string() }) });
const candidates = await page.observe("页面上所有可点击的菜单项");

await session.close();
```

### 5.2 快捷方式:`wrap`

```ts
import { wrap } from 'browser-auto/quick'
const browser = await chromium.launch()
const rawPage = await browser.newPage()
const page = wrap(rawPage, { llm: {...} })
await page.act('...')
```

`wrap` 内部就是 `createSession({ browser: { mode: 'attach-page', page: rawPage } })` 的糖,共用同一套实现。

### 5.3 EnhancedPage 类型

```ts
type EnhancedPage = playwright.Page & {
  act(instruction: string, vars?: Record<string, string>): Promise<void>;
  extract<T>(opts: { schema: z.ZodType<T>; instruction?: string }): Promise<T>;
  observe(instruction: string): Promise<NodeRef[]>;
};
```

实现:用 `Proxy` 把三个方法叠加到原生 Page 上,其它方法透传。**不要继承**,Playwright 的 Page 是接口型对象。

## 6. 必须实现的功能(每条对应一个差异化钩子)

### 6.1 三个核心方法

LLM 规划阶段 **不产出裸 selector**,产出结构化 `ActionStep`:

```ts
type ActionStep =
  | { kind: "click"; target: NodeRef }
  | { kind: "fill"; target: NodeRef; value: string | { var: string } }
  | { kind: "press"; key: string }
  | { kind: "select"; target: NodeRef; value: string }
  | { kind: "hover"; target: NodeRef }
  | {
      kind: "wait";
      condition: "idle" | "visible" | "timeout";
      target?: NodeRef;
      ms?: number;
    };

type NodeRef = {
  role: string; // a11y role
  name: string; // accessible name
  selectors: string[]; // 多候选 fallback 链
};
```

执行阶段按 `selectors` 顺序尝试,任一命中即可。

### 6.2 act 流程(走 L3)

```
1. 计算 fingerprint
2. memory.recallPlan(instruction, fp) → 命中?
   ├ 是 → 重放 → 失败则 invalidate + 走 LLM
   └ 否 → snapshot → planner(LLM) → ActionStep[] → executor
3. 成功后 memory.rememberPlan
```

### 6.3 extract 流程(走 L2)

```
1. 对 schema 每个字段 fieldKey:
   2. semanticKey = `extract:${fieldKey}`
   3. memory.recallLocator(semanticKey, fp) → 命中?
      ├ 是 → 直接 page.textContent / inputValue → 成功就用,空值/异常 → invalidate
      └ 否 → snapshot → planner(LLM) → 找节点 → memory.rememberLocator
4. 组装为 schema 形状返回
```

### 6.4 钩子 1:记忆文件可读可改

参见 4.4.2 的 schema。**文件结构必须文档化**(zod schema + 注释),用户能手动改。

### 6.5 钩子 2:页面指纹

- 用 `page.accessibility.snapshot()` 拿无障碍树
- 提取 role 属于 `[button, link, textbox, combobox, checkbox, menuitem, tab, menu]` 的节点
- 拼接 `role + name + 父级 role 链` 后 sha256
- **不要** 把整个 HTML 喂进去

### 6.6 钩子 3:Snapshot 策略可插拔

```ts
interface SnapshotStrategy {
  name: string;
  getInteractiveNodes(page: Page, region?: NodeRef): Promise<InteractiveNode[]>;
  fingerprint(nodes: InteractiveNode[]): string;
}
```

默认实现 `A11ySnapshotStrategy`。`region` 参数为 v0.3 L1 子树裁剪铺路 —— v0.1 不传或忽略即可,但接口必须有这个参数。

> **为什么这么设计**:v0.5 Ant Design 适配只要写 `AntDesignSnapshotStrategy` 注入,核心不动一行;v0.3 L1 子树推理只要传 region 参数。

### 6.7 钩子 4:Self-healing

- 命中 L3 / L2 后,执行前先做轻量校验(目标 NodeRef 是否仍存在)
- 校验失败 → invalidate 该条 → fallback 到上一层
- 全过程对调用方透明,debug 日志体现:
  - `[memory] L3 hit` / `[memory] L3 stale, fallback to L2` / `[memory] L0 plan, write L3+L2`

### 6.8 钩子 5:Executor 拦截器接口

定义但 **不实现任何拦截器**,接口必须存在:

```ts
interface BeforeActionInterceptor {
  name: string;
  before(ctx: ActionContext): Promise<void | { abort: true; reason: string }>;
}

interface AfterActionInterceptor {
  name: string;
  after(ctx: ActionContext, result: ActionResult): Promise<void>;
}

type ActionContext = {
  sessionId: string;
  page: Page;
  step: ActionStep;
  instruction: string;
};
```

createSession 接收 `policies: BeforeActionInterceptor[]`,executor 在每次执行 step 前后串行调用。

> **为什么这么设计**:v0.4 加域名白名单、脱敏、审计,直接写 interceptor。

### 6.9 钩子 6:结构化 Trace 事件流

定义 `TraceEvent` 类型 + `TraceSink` 接口,v0.1 提供 `ConsoleTraceSink` 和 `FileTraceSink`(JSONL):

```ts
type TraceEvent =
  | { type: 'act.start', sessionId, pageId, instruction, ts }
  | { type: 'memory.hit', sessionId, layer: 'L3'|'L2'|'L1', key, ts }
  | { type: 'memory.miss', sessionId, layer, key, ts }
  | { type: 'memory.invalidate', sessionId, layer, key, reason, ts }
  | { type: 'memory.write', sessionId, layer, key, ts }
  | { type: 'llm.request', sessionId, snapshotSize, model, ts }
  | { type: 'llm.response', sessionId, plan: ActionStep[], usage, ts }
  | { type: 'step.start', sessionId, step, attempt, ts }
  | { type: 'step.success', sessionId, step, selectorIndex, ts }
  | { type: 'step.fail', sessionId, step, error, ts }
  | { type: 'act.end', sessionId, success, durationMs, ts }
  | { type: 'extract.start' | 'extract.end', ... }
  | { type: 'observe.start' | 'observe.end', ... }

interface TraceSink {
  write(event: TraceEvent): Promise<void>
}
```

> **为什么这么设计**:v1.0 业务版"剧本回放"的源数据,现在不收集后面没源头。

### 6.10 钩子 7:Daemon-ready 架构

createSession 的 `browser.mode` 必须支持 `'attach'`(connectOverCDP)。
v0.1 **不写** 长驻进程本身,只保证"能接管已有 Chrome"。
